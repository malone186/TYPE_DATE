## 1. 제품 개요

### 1-1. 제품 비전

> "16번의 소개팅, 그 끝에 진짜 인연이 있다"

16가지 MBTI 유형의 캐릭터와 채팅 형식으로 소개팅을 진행하고,
모든 만남이 끝났을 때 당신의 **진짜 인연 유형**을 알려주는 소개팅 시뮬레이션 앱.

16번 만나고, 16번 선택하고, 16번 느낀다.
그 과정에서 어떤 사람에게 마음이 열리는지, 어떤 순간에 설레는지 자연스럽게 드러난다.
결말에서 밝혀지는 건 MBTI가 아니라 — **당신이 진짜 끌리는 인연의 유형**이다.

### 1-2. 핵심 차별점

| 경쟁 앱 | TYPE DATE |
|---|---|
| 단순 소개팅 게임 | 16화 완료 시 진짜 인연 유형 도출 → 결말이 있는 스토리 |
| 선택지가 뻔함 | ABCD 4지선다 + 이중 구조 → 고를 때마다 진짜 고민됨 |
| 결과가 텍스트 | SNS 공유용 카드 → 바이럴 구조 |
| 1회성 사용 | 16화 순차 해금 → 장기 플레이 |
| 대화가 퀴즈처럼 끊어짐 | 도입부부터 마지막 턴까지 하나의 채팅 스레드로 계속 누적 |

### 1-3. 타깃

- **주 타깃:** 20대 MBTI 관심 있는 남녀
- **부 타깃:** 소개팅 경험 있는 직장인 20~30대

### 1-4. 수익 모델

| 단계 | 조건 | 내용 |
|---|---|---|
| 1단계 | 출시 초기 | 완전 무료, 광고 없음, 16화 전부 무료. 바이럴 집중 |
| 2단계 | 다운로드 1만+ | 배너 광고 추가 / 광고 제거 구매 2,200원 |
| 3단계 | 콘텐츠 확장 | 여자 캐릭터 라인 무료 유지 / 남자 캐릭터 라인 확장팩 유료 |

## 3. 유저 페르소나

### 페르소나 A — 김지훈 (22세, 대학생)
- 소개팅 직접 나가기는 부담. 근데 인연은 만나고 싶음
- "어떤 유형이랑 잘 맞는지 모르겠다"는 막연한 고민 있음
- SNS에서 MBTI 관련 짤 자주 공유. 결과 카드 공유 욕구 높음
- 친구가 공유한 카드 보고 설치하는 전형적 유입 경로

### 페르소나 B — 이수현 (26세, 직장인)
- 퇴근 후 5~10분 단위로 즐길 수 있는 콘텐츠 원함
- 소개팅 경험 있지만 "왜 안 맞았는지" 여전히 모름
- 결과 보고서에서 내 소개팅 스타일 보고 지인 공유

```

[이름 입력]
    "상대가 당신을 뭐라고 부르면 좋을까요?"
    이름/닉네임 입력 → 이후 대사에서 "OO씨"로 반영

[프롤로그]
    카톡 형식 (민준 대화 → 앱 첫 실행 온보딩 → 배정 알림)

[캐릭터 선택 화면]
```

### 5-2. 화면별 주요 스펙

**소개팅 채팅 화면**
- 도입부: 지수가 먼저 도착해 인사·주문·착석하는 과정을 선택지 없이 자동 재생 (타이핑 인디케이터 포함) → 자연스럽게 턴1로 연결
- 배경: 카페 사진(`assets/images/background.png`) 위에 테마 배경색 스크림(라이트 35% / 다크 55% 불투명도)을 얹어 실제로 만난 장소 느낌은 유지하면서 텍스트 가독성 확보 (v1.6). **(v1.7)** 소개팅 채팅 화면에는 아이폰 상태바를 표시하지 않는다 (배경 사진 몰입 유지).
- 헤더: 뒤로가기 + 캐릭터 프사 + 이름, **(v1.7)** 이름 아래에 `MBTI · 나이` 부제(예: `ENFP · 26`)를 함께 표기
- 상대 말풍선: 왼쪽 정렬, 캐릭터 프사 옆. 메시지 길이에 비례한 "입력 중..." 표시 후 노출 (카카오톡 대화와 같은 리듬)
- 주인공 독백: **(v1.7)** 별도 말풍선 없이 회색 이탤릭 텍스트로 중앙 정렬 — 실제 메시지(말풍선)와 시각적으로 명확히 구분하면서 화면을 가볍게 유지. (v1.6의 라벤더 유리질 버블 + "속마음" 라벨은 배경/글자 가독성 피드백을 반영해 제거)
- 선택지 버튼: 화면 하단 고정 바, 배경 투명(테두리만), A/B/C/D 레이블 + 선택지 텍스트
- 선택 후: 호감도 이펙트(테두리 플래시 + 파편) → "입력 중..." 표시 → 상대 반응 말풍선 → 다음 턴 자동 진행
- 대화 기록: 완료된 턴들이 계속 누적되어 스크롤로 다시 볼 수 있음
- 진행 바: 상단에 13턴 중 현재 위치 표시

**소개팅 클로징 씬 (v1.5)**
- 13턴이 끝나면 새 화면으로 전환하지 않고, 같은 채팅 스레드 맨 아래에 이어서 짧은 마무리 연출이 자동 재생됨 (도입부와 동일한 타이핑 리듬)
- 결과(Ending: 성공/친구/실패)에 따라 서로 다른 대본이 재생 — 카페를 나서는 장면 묘사 + 지수의 마지막 인사 + 주인공 속마음
- 마지막 줄까지 다 나오면 화면 하단에 "결과 확인하기" 버튼이 나타남 (선택지 버튼과 같은 위치/스타일). 자동으로 넘어가지 않고 버튼을 눌러야 결과 보고서 화면으로 이동

**전 화면 공통**
- 테마 토글 버튼을 모든 화면 상단(앱바 또는 화면 우상단)에 배치. 기본값은 라이트 모드
- **(v1.7)** 앱 전역 배경을 라벤더 → 퍼플 → 핑크 대각선 그라디언트로 통일 (좌상단 페리윙클 → 중앙 라벤더 → 우하단 핑크, 쿨톤 글로우 블롭 포함). 기존 새벽빛 크림/코랄 톤 대신 라벤더 계열로 배경 컬러 토큰(bg/border)까지 이동. 라이트/다크 각각의 퍼플 버전 제공. 소개팅 채팅 화면은 카페 사진 배경 유지(그 위 스크림도 라벤더 톤)
- **(v1.7)** 아이폰 상단 상태바 모방 위젯(`PhoneStatusBar`): 좌측에 현재 시각(분 단위 자동 갱신), 우측에 신호·데이터(Wi-Fi)·배터리 잔량(퍼센트 + 채워진 아이콘, 충전 상태 아님). 색상은 테마에 따라 자동 대응. **스플래시/타이틀, 소개팅 상대 선택, 프롤로그 민준 대화 화면에만** 적용하고 소개팅 채팅 화면과 에필로그 민준 카톡에는 넣지 않는다
- **(v1.7)** 스플래시/타이틀 화면 로고를 흰 배경이 박힌 이미지 대신 투명 배경 로고(`assets/images/logo_mark.png`)로 교체해 배경 그라디언트 위에 로고만 떠 보이게 함

## 6. 게임 로직 명세

### 6-1. ABCD 이중 축 시스템

각 턴마다 4개 선택지, 선택 1회 → **2개 축 점수 동시 기록**.

```
턴 구조:
  Primary축 × Secondary축 = 4가지 조합

  선택 A: Primary-① + Secondary-①
  선택 B: Primary-① + Secondary-②
  선택 C: Primary-②  + Secondary-①
  선택 D: Primary-②  + Secondary-②
```

- 13턴 × 2축 = **총 26개 데이터 포인트**

**궁합 축 비교 그리드 (v1.6 추가)**

궁합 항목이 별점과 한 줄 코멘트만으로는 "어떤 점이 맞고 어떤 점이 다른지" 알기 어렵다는 피드백을 반영해, MBTI 4개 축(에너지 방향 E/I, 정보 해석 방식 N/S, 판단 기준 T/F, 생활 패턴 J/P)마다 "나"와 상대방의 성향 글자를 나란히 보여주고 일치/차이 배지를 붙인 비교 행을 추가했다. 상단에는 "나와 지수, 4개 성향 중 N개 일치" 형태로 전체 요약을 표시한다. 나의 성향 글자는 실제 플레이 중 축적된 `axisScore`로 계산된 `axisLetters`를, 상대 성향 글자는 캐릭터의 고정 `mbti` 값을 그대로 사용해 축 순서(E/I·N/S·T/F·J/P)를 1:1로 비교한다.

## 10. 데이터 모델

```dart
// 캐릭터
class TDCharacter {
  String id;            // 'date01_jisu'
  String name;          // 지수
  int age;              // 26
  String job;           // 콘텐츠 기획자
  String location;      // 인천
  String mbti;          // ENFP
  String intro;         // 한 줄 소개
  List<String> tags;    // 관심사 태그
  bool isUnlocked;      // 해금 여부
  String? imagePath;    // 프사 이미지 경로
}

// 카톡 모방 채팅 한 줄 (프롤로그/에필로그/소개팅 도입부·클로징 씬용)
class ChatLine {
  String sender;         // 'me' 또는 캐릭터 이름 / 'system'
  String text;
  bool isSystemNote;     // 화면 설명 — 가운데 정렬 안내문
  bool isMonologue;      // 주인공 속마음 — 채팅 버블 + "속마음" 라벨
}

// 결과(Ending)별 소개팅 클로징 씬 대본 반환 (v1.5)
// List<ChatLine> closingScriptFor(Ending ending)

// 선택지 (2개 축 동시 기록)
class Choice {
  String label;           // A/B/C/D
  String text;            // 선택지 텍스트
  String primaryAxis;     // 'E', 'I', 'N', 'S', 'T', 'F', 'J', 'P'
  String secondaryAxis;   // 'E', 'I', 'N', 'S', 'T', 'F', 'J', 'P'
  int likeScore;          // +1 또는 -1
  String npcReaction;     // 상대방 반응 텍스트
}

// 턴
class Turn {
  int turnNumber;         // 1 ~ 13
  String npcMessage;      // 상대 메시지 ("{name}씨" 토큰 포함 가능)
  String monologue;       // 주인공 내면 독백
  bool isPlayerInitiated; // 주인공 질문형 여부
  List<Choice> choices;   // A, B, C, D
}

// 소개팅
class BlindDate {
  String id;                     // 'date01'
  TDCharacter character;
  List<Turn> turns;              // 13턴
  List<ChatLine> openingScript;  // 도입부(도착·인사·착석) — 선택지 없이 자동 진행
}

// 소개팅 결과
class DateResult {
  String dateId;                    // 'date01'
  int likeScore;                    // 호감도 합계 (-13 ~ +13)
  Ending ending;                    // success / friend / fail
  Map<String, int> axisScore;       // {'E':4, 'I':2, 'N':5, 'S':1, 'T':3, 'F':4, 'J':2, 'P':5}
  String styleType;                 // 'EF' / 'ET' / 'IF' / 'IT'
  DateTime completedAt;
}

// 전체 진행 상황
class GameProgress {
  Map<String, DateResult> results;  // dateId → DateResult
  int totalCompleted;               // 완료한 소개팅 수
  String? overallStyle;             // 전체 스타일 유형 (16화 완료 시)
}
```

콘텐츠 파일 구조(구현 기준): 캐릭터/데이터 조립은 `episode1_data.dart`, 실제 대사·연출 대본(프롤로그/도입부/13턴/에필로그)은 `episode1_script.dart`로 분리해 대본만 수정할 때 다른 파일을 건드리지 않도록 했다.
