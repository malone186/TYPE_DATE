# 대본 오타 · 대화 자연스러움 검수리스트

> 대상: `src/data/male/episode*.ts` 대사 텍스트 전반
> 방식: **한 줄씩 소리 내어 읽으며** 교정. 턴 순서대로.
> 사용법: 대사 블록(npcMessage / choice.text / npcReaction / monologue / opening·closing)마다 통과. `[ ]` → `[x]`

---

## A. 오타 · 맞춤법

### A-1. 자주 틀리는 맞춤법
- [ ] `되/돼` — "안 돼요"(O) / "됐어요"(O) / "되요"(X)
- [ ] `안/않` — "안 해요"(부사) / "하지 않아요"(용언)
- [ ] `-데/-대` — 직접경험 "좋던데" / 전달 "좋대"
- [ ] `-든/-던` — 선택 "가든 말든" / 과거 "갔던"
- [ ] `-로/-으로` — 받침 유무에 맞게 ("각도로" / "손으로")
- [ ] `-率/-율` 류 두음, `며칠`(O)/`몇일`(X), `왠지`(O)/`웬지`(X)
- [ ] 어미 `-ㄹ게요`(O) / `-ㄹ께요`(X) — "말할게요"

### A-2. 띄어쓰기
- [ ] `{name}씨` / `{name} 씨` 표기가 **에피소드 내내 통일** (기존 대본은 붙여쓴 `{name}씨` 우세 — 혼용 금지)
- [ ] 캐릭터 호칭 `태윤씨`·`하준 씨` 등 띄어쓰기 방식이 파일 내 일관
- [ ] 의존명사 띄어쓰기 — "할 수 있다", "간 지 오래", "먹을 것"
- [ ] 보조용언 — "봐줘요", "해두죠" 등 붙임/띄움이 파일 내 일관
- [ ] 숫자+단위 — "삼 년", "8만", "여섯 시" 표기 방식 일관 (한글/아라비아 혼용 규칙 통일)

### A-3. 조사 · 문장부호
- [ ] 조사 `은/는`, `이/가`, `을/를`, `와/과` 받침에 맞게
- [ ] 마침표·물음표·느낌표 누락/중복 없음 (대사 끝 부호 스타일 일관)
- [ ] 쉼표 위치가 호흡에 맞음 (한 문장에 과도한 쉼표 없음)
- [ ] 따옴표(작은/큰) 짝 맞음, 문자열 이스케이프 깨짐 없음

### A-4. 치환자 · 고유명사
- [ ] `{name}` 정확히 소문자·중괄호 (`{Name}`·`{ name }`·`(name)` 오타 없음)
- [ ] 상대 캐릭터 이름이 **처음부터 끝까지 동일** (오프닝·턴·클로징·에필로그 전부)
- [ ] 지명·직업·구독자 수 등 숫자 정보가 여러 곳에서 **모순 없음** (예: 나이 27 ↔ 30)
- [ ] MBTI 표기 대문자 4자 일관

---

## B. 대화 자연스러움

### B-1. 흐름 연결 (핵심)
- [ ] `npcMessage`(상대 얘기) → `choice.text`(내 반응)가 **말이 이어짐** (동문서답 X)
- [ ] `choice.text` → `npcReaction`(상대 반응)이 그 선택을 **실제로 받아친** 반응
- [ ] `monologue`(속마음)가 바로 위 npcMessage 상황과 맞물림
- [ ] 이전 턴 → 다음 턴 화제 전환이 자연스러움 (뜬금없는 점프 X)
- [ ] `playerPrompt`(내 질문) → `npcMessage`(상대 답) 문답이 어긋나지 않음

### B-2. 말투 · 톤 일관성
- [ ] 상대 캐릭터 말투(존댓말/반말·어미 습관)가 전 대사에서 흔들리지 않음
- [ ] 주인공 선택지 4개의 말투가 서로 자연스럽게 통일 (한 사람 입에서 나온 말)
- [ ] 전체 **담백한 톤** 유지 — 과장 감탄·오글거리는 미사여구 없음
- [ ] npcReaction이 캐릭터 성격과 맞음 (ISTJ가 갑자기 오버 X)

### B-3. 사람이 실제로 하는 말인가
- [ ] 소리 내어 읽었을 때 **혀에 걸리는 문장**(비문·꼬인 어순) 없음
- [ ] 한 대사가 지나치게 길어 숨차지 않음 (필요시 문장 분리)
- [ ] 번역투/설명체("~라는 것이다") 대신 구어체
- [ ] 같은 단어·표현이 짧은 구간에서 반복되지 않음
- [ ] 감정 표현이 상황 대비 과하거나 부족하지 않음

### B-4. 내용 정합성
- [ ] 상대가 방금 한 말을 선택지가 헛짚지 않음 (안 한 말에 반응 X)
- [ ] npcReaction이 `likeScore`(+1 호감 / -1 비호감) 방향과 톤이 맞음
- [ ] 정답 선택지(+1)의 npcReaction이 부정적으로 읽히지 않음 (반대도 동일)
- [ ] 오프닝에서 깐 설정(음료·자리·직업병 등)이 본편에서 모순 없이 회수
- [ ] 클로징·에필로그가 실제 오간 대화 내용과 어긋나지 않음

---

### 교정 팁
- npcMessage → 선택지 A~D → 각 npcReaction을 **한 세트로 묶어** 위→아래로 읽으면 연결 어색함이 가장 잘 보임
- 오타는 눈으로, 자연스러움은 **입으로**. 두 번 나눠서 훑기
