3. 개발 중 발생 오류 및 해결 내용
【오류 3-1】 Riverpod 3.x API 불일치로 인한 컴파일 에러

현상  : StateNotifier, StateNotifierProvider, StateProvider가 undefined 에러 발생.

원인  : 설치된 flutter_riverpod 3.3.2는 구버전 StateNotifier API를 제거하고 Notifier / NotifierProvider 체계로 대체. StateProvider는 legacy.dart로 이동.

해결  : game_state.dart를 Notifier<T> / NotifierProvider.autoDispose<T> 패턴으로 재작성. 테마 전환용 StateProvider는 legacy.dart import 유지. 이후 flutter analyze 이슈 0건 확인.

【오류 3-2】 한글 텍스트 빈 네모(tofu) 렌더링

현상  : Flutter Web(CanvasKit)에서 한글 글자가 일시적으로 빈 네모 박스로 표시.

원인  : Pretendard 폰트를 번들링하지 않고 시스템 기본 폰트에 의존. CanvasKit이 없는 글리프를 Google 서버에서 런타임 fetch하는 동안 렌더링 지연 발생.

해결  : Pretendard v1.3.9 OTF 4종(Regular/Medium/SemiBold/Bold)을 assets/fonts/에 번들링하고 pubspec.yaml의 fonts: 섹션 및 ThemeData(fontFamily: 'Pretendard')로 등록. 이후 네트워크 없이 즉시 정상 렌더링 확인.

【참고 3-3】 Chrome 창 가림 시 타이머 쓰로틀링 (검증 환경 한정)

현상  : 자동화 검증 중 소개팅 채팅 화면의 자동 전환 타이머(1.1~1.3초)가 멈추는 현상.

원인  : 검증용 Chrome 창이 다른 창에 가려질 때 Chrome이 해당 탭을 백그라운드로 간주해 JS 타이머를 강제 쓰로틀링. 실사용 환경에서는 발생하지 않음.

조치  : 코드 수정 없음 — 앱 자체 정상. 검증 환경 주의사항으로만 기록.

【참고 3-4】 폴더 재배치 후 빈 디렉터리 삭제 실패 (해결 보류)

현상  : 파일 이동 후 빈 type_date_app/ 폴더 삭제 시 "다른 프로세스가 사용 중" 오류.

원인  : VSCode Dart Analysis Server가 기존 경로 핸들을 유지하고 있음. 파일 이동 자체는 정상 완료.

조치  : 에디터 재시작 후 삭제 가능. 코드·데이터 영향 없음.

【오류 3-5】 UX 구조가 PRD 의도와 다르게 구현됨 (수정 완료)

현상  : ① 민준/에필로그 채팅이 탭해야만 다음 메시지 출력. ② 소개팅 채팅이 턴마다 화면 리셋되는 "퀴즈 형태"로 보임.

원인  : PRD 문구 "탭하면 다음 메시지 출력"을 글자 그대로 구현. 실제 의도(실제 만남처럼 자연스러운 흐름)와 괴리.

해결  : 전면 재작업 완료.
① kakao_chat_view.dart — 메시지 길이에 비례한 딜레이(0.7~2.2초)로 자동 진행.
② blind_date_chat_screen.dart + game_state.dart — 선택한 답변이 실제 말풍선으로 쌓이는 메신저형 스레드 구조로 변경.
