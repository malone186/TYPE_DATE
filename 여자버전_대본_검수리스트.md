# 여자버전 대본 검수리스트

> 대상: `src/data/male/episode*.ts` (주인공 = **여성 27세**, 상대 = **남성 캐릭터**)
> 기준: `src/types/index.ts` 데이터 모델 + 대본 작성 컨벤션
> 사용법: 에피소드 1개당 이 리스트를 한 번씩 통과. `[ ]` → `[x]`

---

## 0. 검수 전 확인
- [ ] 파일 상단 주석에 캐릭터명 / MBTI / 직업 / **턴 수**가 명시돼 있다
- [ ] 주석의 턴 수 = 실제 `turns` 배열 길이 (※ 타입 주석엔 "13턴"이라 적혀 있으나 실데이터는 9턴 — 실제 배열 기준으로 검수)
- [ ] `import { BlindDate, ChatLine } from '../../types';` 경로 정상

## 1. 메타 / 네이밍 규칙
- [ ] `id` = `male_<mbti소문자>` (예: `male_enfp`)
- [ ] `character.id` = `male_<mbti>_<이름로마자>` 형식
- [ ] export 변수명 = `maleEpisode<Mbti>` (예: `maleEpisodeEnfp`)
- [ ] `imagePath` = `assets/images/<MBTI대문자>_male.png`
- [ ] `facePath` = `assets/images/<MBTI대문자>_male_face.png`
- [ ] 이미지 파일이 실제 `assets/images/`에 존재

## 2. 캐릭터 카드 (`character`)
- [ ] `name` / `age` / `job` / `location` / `mbti` 모두 채워짐
- [ ] `mbti`가 파일명·id와 일치 (오타·불일치 없음)
- [ ] `intro` 한 문장이 해당 MBTI 성격을 드러냄
- [ ] `tags` 5개, 모두 `#`로 시작, 캐릭터 설정과 일치
- [ ] `isUnlocked` 값이 의도대로 설정됨
- [ ] 나이·직업이 대본 본문 내용과 모순 없음

## 3. 도입부 (`openingScript`)
- [ ] 도착 → 인사 → 착석 흐름이 자연스럽게 자동 진행됨
- [ ] `sender`가 `system`(장소설명) / 캐릭터명 / `me` 중 하나로 정확
- [ ] 장소 설명 줄에 `isSystemNote: true`
- [ ] 주인공 속마음 줄에 `isMonologue: true`
- [ ] 도입부 장소·분위기가 `closingScripts` 배경과 일관됨

## 4. 턴 공통 (`turns[]`)
각 턴마다:
- [ ] `turnNumber`가 1부터 빠짐/중복 없이 순서대로
- [ ] `npcMessage`는 **상대가 자기 얘기를 털어놓는** 내용 (인터뷰형 질문 X)
- [ ] `monologue`(주인공 속마음)가 npcMessage에 대한 관찰로 존재
- [ ] `isPlayerInitiated: true`인 턴은 `playerPrompt`(내 질문)가 반드시 있음
- [ ] `isPlayerInitiated: false`인 턴은 `playerPrompt`가 없음/불필요
- [ ] 선택지는 정확히 **4개** (`A` / `B` / `C` / `D`)

## 5. 선택지 규칙 (`choices[]`)
각 선택지마다:
- [ ] `label`이 A/B/C/D 순서대로
- [ ] `text`는 **상대 얘기에 대한 반응** (되묻는 인터뷰 X)
- [ ] `primaryAxis` / `secondaryAxis`가 MBTI 8축(E,I,N,S,T,F,J,P) 문자
- [ ] `likeScore`는 `+1` 또는 `-1` 만 사용
- [ ] `npcReaction`이 해당 선택에 맞는 반응으로 존재

턴 단위 밸런스:
- [ ] 한 턴의 4개 선택지 `likeScore` = **+1 두 개, -1 두 개** (2:2 균형)
- [ ] 한 턴의 4개 선택지가 하나의 **축 쌍**을 A/B/C/D로 커버 (예: E-I × T-F)
- [ ] 정답(+1)/오답(-1) 판정이 캐릭터 성격상 납득 가능

## 6. 축 커버리지 (에피소드 전체)
- [ ] E/I, N/S, T/F, J/P **4개 축 모두** 여러 턴에 걸쳐 등장
- [ ] 특정 축만 과도하게 몰리지 않음 (엔딩 유형 EF/ET/IF/IT 계산이 유효하도록 E-I·T-F 축이 충분히 반영)
- [ ] 축 판정이 애매한 선택지 없음 (한 선택지 = 명확한 축 방향)

## 7. 톤 앤 매너
- [ ] 전체적으로 **담백한 톤** 유지 (과장·오글거림 없음)
- [ ] npcReaction이 캐릭터 말투로 일관 (반말/존댓말 톤 흔들림 없음)
- [ ] 대사에 시대·상황 안 맞는 표현/비속어 없음
- [ ] 상대 남성 캐릭터의 매력 포인트가 대본 안에서 드러남

## 8. 클로징 (`closingScripts`)
- [ ] `success` / `friend` / `fail` **3종 모두** 존재
- [ ] 각 엔딩 톤이 명확히 구분됨 (성공=설렘 / 친구=담백 / 실패=아쉬움)
- [ ] 각 클로징에 주인공 마무리 속마음(`isMonologue`) 포함
- [ ] 결말이 앞선 9턴 서사와 논리적으로 연결됨

## 9. 유형별 리포트 (`styleInfo`)
- [ ] `EF` / `ET` / `IF` / `IT` **4종 모두** 존재
- [ ] 각 코드의 `code` 값이 키와 일치
- [ ] `emoji` / `title` / `summary` 채워짐, 유형 특성과 맞음
- [ ] `goodPoint` / `badPoint`가 이 캐릭터 기준으로 구체적 (일반론 X)
- [ ] `compatibilityStars` = ★/☆ 5칸 형식
- [ ] `compatibilityComment`가 궁합을 납득되게 설명
- [ ] `endingMessages`에 success/friend/fail **3종 모두** 존재

## 10. 조력자 에필로그 (해당 시)
- [ ] 조력자 친구 라인(`maleEpilogue<Mbti>Lines`)이 `ChatLine[]`로 정상
- [ ] 근황·결과 카드 흐름이 자연스러움
- [ ] 다음 소개팅 유도 멘트(16명 채우기 등) 포함

## 11. 치환자 / 기술 검증
- [ ] 주인공 이름 자리에 `{name}` 치환자 정확히 사용 (오타 `{Name}`·누락 없음)
- [ ] 큰따옴표/작은따옴표로 감싼 문자열 내 따옴표 이스케이프 정상 (예: `job: "'태윤약국'..."`)
- [ ] `src/data/male/index.ts`에 이 에피소드가 등록됨
- [ ] `npx tsc --noEmit` 타입 에러 없음
- [ ] 줄바꿈·특수문자로 인한 JSON/TS 파싱 오류 없음
